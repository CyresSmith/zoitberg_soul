# Daily Log: 2026-04-15 (Update 2)

## Auth Refactoring
- **Issue:** TypeScript error `This expression is not callable` on `NextAuth`.
- **Root Cause:** NextAuth v5 beta.25 in a Next.js 16 environment requires a very specific export/import pattern. Direct usage of `NextAuth()` can fail if types are mismatched between peer dependencies.
- **Fix:** 
    - Verified `next-auth@5.0.0-beta.25` usage.
    - Updated `auth.ts` to use explicit async callbacks and ensured the object passed to `NextAuth` strictly matches the expected `NextAuthConfig` interface.
    - Added `trustHost: true` (mandatory for Vercel/Next.js 16).
    - Ensured `Credentials` provider is imported from `next-auth/providers/credentials`.

## Knowledge Base Update
- Documented that for Next.js 16 + NextAuth v5, `trustHost: true` is not optional, it's a requirement to prevent "Untrusted Host" errors on Vercel.
